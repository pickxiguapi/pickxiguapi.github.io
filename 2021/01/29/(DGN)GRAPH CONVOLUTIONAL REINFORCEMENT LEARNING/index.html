GRAPH CONVOLUTIONAL REINFORCEMENT LEARNING（ICLR2020）

论文源代码

ABSTRACT

  • 多智能体环境一直动态变化使得学习合适的合作策略，学习agent之间相互作用的抽象表示更加困难
  • 使用图卷积适应多智能体环境下动态变化的底层图，使用关系核(relation kernel)捕捉agent之间相互作用

INTRODUCTION

DGN is graph convolutional reinforcement learning, where the multi-agent environment is modeled as a graph, each agent is a node, and the encoding of local observation of agent is the feature of node. We apply convolution to the graph of agents. By employing multi-head attention as the convolution kernel, graph convolution is able to extract the relation representation between nodes and convolve the features from neighboring nodes just like a neuron in a convolutional neural network (CNN). Latent features extracted from gradually increased receptive fields are exploited to learn cooperative policies. Moreover, the relation representation is temporally regularized to help the agent develop consistent cooperative policy.

多智能体环境被建模成图，每个agent是一个节点，agent的局部观察编码为节点特征，节点自然和邻居构成联系。通过在卷积核上应用多头注意力，提取节点之间的关系表示，并将特征与相邻节点进行卷积，提取潜在特征，学习合作策略。

METHOD

把multi-agent environmen建模为图，智能体为节点iii，每个节点有一组邻居Bi\mathbb{B_i}Bi​

邻居如何确定？： 由距离或者其他环境的指标

图卷积的一个优势： 只需要使用局部通信，但是因为图卷积可以逐渐增加图的接受域(recepetive field)，所以实际合作到的可能有二阶邻居三阶邻居等等，合作范围没有受限

需要注意的是，多智能体学习中环境是动态的，所以构成的图也是动态变化的，然而GCN中的cora数据集是静态图，所以算法必须要能够适应动态的图

GRAPH CONVOLUTION 图卷积

DGN由三种模块组成：

  • observation encoder 观测编码器
  • convolutional layer 卷积层
  • Q network Q网络

由局部观测信息编码成特征向量，由MLP/CNN输入，使用relational kernel经过图卷积层生成融合了合作信息的新特征向量，最后输入Q network。

最大的困难：agent的数量和位置一直变化，图也在不断的变化，给图卷积带来了困难。为了解决该问题，我们将所有agent在ttt时刻的特征向量按index合并在一个N×LN \times LN×L的特征矩阵FtF^tFt中，其中N为agent的数量，L为特征向量的长度。使用(∣Bi∣+1)×N(|\mathbb{B}_i|+1)\times N(∣Bi​∣+1)×N的邻接矩阵CitC_i^tCit​从中取得需要的特征，第一行是agent iii的index one-hot编码，剩下是邻居的one-hot编码。通过Cit×FtC_i^t \times F^tCit​×Ft获得节点i的特征向量。

模仿Densenet：每一层的特征都串联拼接传递到最后的Q网络中，从不同的接受域中提取特征

RELATION KERNEL 关系核

卷积核可以整合接受域的特征并提取潜在的特征，但是有一个重要的性质是核应当独立于输入特征向量的顺序。

改进自RRL（原文：Relational Deep Reinforcement Learning）本文中关系卷积核使用多头点积注意力机制计算agent之间的交互。对于每个注意头，所有输入特征的值表示由关系加权并相加，然后将agent i的M个注意头输出串联输入激活函数，产生卷积层输出。

hi′=σ( concatenate [∑j∈B+iαijmWVmhj,∀m∈M])h_{i}^{\prime}=\sigma\left(\text { concatenate }\left[\sum_{j \in \mathbb{B}_{+i}} \alpha_{i j}^{m} \mathbf{W}_{V}^{m} h_{j}, \forall m \in \mathbf{M}\right]\right) hi′​=σ⎝⎛​ concatenate ⎣⎡​j∈B+i​∑​αijm​WVm​hj​,∀m∈M⎦⎤​⎠⎞​

多头注意力机制可以使核独立于输入特征向量的顺序，同时更多的注意头给出更多的关系表示，使训练在经验上更加稳定。通过多个卷积层提取更高阶的关系表示，捕捉agent之间的相互作用。

TEMPORAL RELATION REGULARIZATION 时间关系正则化

使用KL散度解决合作关系变化剧烈的问题
许多合作任务主要agent之间保持长期持续的合作才能获得最终的回报。为了使学习到的多头注意力权重分布在时间步上稳定，本文使用了时间关系正则化。以下一状态下注意权重分布作为当前注意权重分布的目标，采用KL散度来衡量当前的注意权重分布与目标注意权重分布的差异，最小化该KL散度则为鼓励agent行程一致的关系表示，从而形成一致的合作关系。

Note: 计算KL散度的网络不是taget network而是current network，因为关系表示与特征提取的权值高度相关，需要使用最新的网络关系表示

加上KL正则化的损失修正如下：

EXPERIMENTS

实验使用MAgent平台中的三个环境battle和jungle和routing，网格为11*11，进行离散的行动或者是攻击。
除了和其他算法进行效果对比，同时还进行了消融实验，评估DGN-R(without temporal relation regularization)和DGN-M(further DGN-R with mean kernels instead of relation kernels)

BATTLE

环境中有N个agent对抗L个更加强大的enemy。agent可以在4个相邻网格中移动或者攻击。但是敌人有更大的移动范围和攻击范围。agent/enemy生命值均为6，enemy使用预先训练的DQN策略，因为单一enemy比agent强，所以agent必须联合对抗enemy。
统计指标为平均奖励、击杀数、死亡数等。实验中N=20 L=12。
图为奖励曲线：

消融实验

DGN-R(without temporal relation regularization)：DGN的行为更加一致和同步，而DGN-R容易被干扰，没有持续合作
DGN-M(further DGN-R with mean kernels instead of relation kernels)：弱于DGN-R，说明图卷积确实可以提取出关系和隐藏特征

JUNGLE

环境中有N个agent和L个食物。agent通过吃食物获得reward，但是攻击其他agent可以获得更高的reward，每个agent可以移动或者攻击。为了验证agent是否可以通过学习协作共享资源，而不是互相攻击。实验中使用N=20以及L=12。DGN可以正确的学习合作，产生较少的攻击，食物可以被周围agent合理分配。奖励曲线比较如下：

CONCLUSION

提出了图卷积强化学习（DGN）：

  • 适应图的动态变化
  • 利用多头点积注意力关系核的卷积提取潜在特征学习合作策略
  • agent之间的关系使用KL临时正则化，让图的变化不那么剧烈，使合作更加一致
