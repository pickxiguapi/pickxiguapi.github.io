Modeling Others using Oneself in Multi-Agent Reinforcement Learning（ICML 2018）

“what would be my goal if I had acted as the other player had?”
如果我是你我会怎么做？

Motivation

在多智能体系统中，对其他智能体的意图进行推理并能够预测它们的行为是很重要的，即对手建模。

本文提出了SOM（Self Other Modeling），显式预测对手的目标，用自己的策略网络基于对手的state预测对手的action，优化对目标的预测。

Note:主要适用于每个agent在回合结束得到的reward依赖于两个agent，在这种环境下agent想要取得最优策略必须同时考虑这两个目标。

Approach

SOM的网络结构如下：

  • 输入：自己的state，自己的goal，预测的对手的goal
  • 输出：自己的策略分布（动作概率分布）和V值

[πiVi]=fi(ssel fi,zself i,z~other i;θi)\left[\begin{array}{c} \pi^{i} \\ V^{i} \end{array}\right]=f^{i}\left(s_{\text {sel } f}^{i}, z_{\text {self }}^{i}, \tilde{z}_{\text {other }}^{i} ; \theta^{i}\right) [πiVi​]=fi(ssel fi​,zself i​,z~other i​;θi)

但是对一个环境，需要用两个相同的SOM网络结构，这两个网络的使用方法不同:

fself (sself ,zself ,z~other ;θself )(1)fother (sother ,z~other ,zself ;θself )(2)\begin{array}{c} f_{\text {self }}\left(s_{\text {self }}, z_{\text {self }}, \tilde{z}_{\text {other }} ; \theta_{\text {self }}\right) (1)\\ \\ f_{\text {other }}\left(s_{\text {other }}, \tilde{z}_{\text {other }}, z_{\text {self }} ; \theta_{\text {self }}\right) (2) \end{array} fself ​(sself ​,zself ​,z~other ​;θself ​)(1)fother ​(sother ​,z~other ​,zself ​;θself ​)(2)​

action mode: 前馈用于选择动作——结合自己的state和对手目标的推断

inference mode: 输出对手的goal_vector，推断对手的goal，然后使用交叉熵优化

每个agent，在agent mode作为自己的输入，在inference mode作为对手的输入

Experiment

TRUE-OTHER-GOAL (TOG): 提供了正确的goal

NO-OTHER-MODEL (NOM)：去掉推断zotherz_{other}zother​的阶段，没有goal vector

INTEGRATED-POLICY-PREDICTOR (IPP)：从NOM出发，但是多一个输出层，也就是预测其他agent的action

SEPARATE-POLICY-PREDICTOR (SPP)：传统opponent modeling

  • 实验环境1：coin 有两个agent a1和a2，他们会分别在环境中收集不同颜色的硬币，每个agent收集一种颜色的硬币，环境中一共有三种颜色的硬币，每个agent会随机给定一种颜色的硬币，当agent收集到不属于他们中任何一个人所需要的硬币时，将会受到惩罚。
  • 实验环境2：recipe 竞争环境
  • 实验环境3：door 合作环境，其中一个agent站在阀门上，才会开门，另一个agent进入获得reward

TOG最大，其次是SOM

使用不同策略进行竞争 TOG > SOM
