摘要

本文探索了针对多智能体领域的深度强化学习方法。首先分析在多智能体情况下传统算法的困难：Q-learning受到环境的非平稳性影响，策略梯度受到随着智能体数量增长而增加的方差的影响。然后本文提出了一种考虑了其他智能体行动策略并能成功学习到需要复杂多智能体协同的actor-critic方法变种。此外，我们介绍了一种训练规则，继承了各个智能体的策略来学习一个更鲁棒的多智能体协同策略。本文展示了在合作和竞争情况下，提出的方法和现有方法相比的优势：在这种环境下，智能体群体能够发现各种物理和信息层面上的合作策略。

引言

强化学习近来被应用在解决许多挑战性问题上去，但是大多数RL的成功应用都集中在单智能体场景，此时无须建模和预测环境中的其他行为体。但是有很多重要的应用场景牵涉到多智能体之间的交互，此时问题也会变得更加复杂，例如多个机器人的控制，多玩家游戏，社会困境的分析等。成功的把强化学习扩展到多智能体环境对于建造能够与人类交互的智能体系统非常重要。

但是传统强化学习算法例如Q-learning或者PG算法都很不适用于多智能体环境。一个问题是在训练过程中每个智能体都在变化，从单一智能体的角度看环境变得不稳定（non-stationary）。这给训练稳定性带来了挑战，并且阻碍了直接利用experience replay。另一方面，PG算法在多智能体协作的时候通常会有很高的方差。

本文提出了一个通用的多智能体学习算法：

  • 学习到的策略在执行时只使用本地信息（their own observations）
  • 不假设环境动力学的可微模型或者智能体之间的任何通信结构（隐式建模）
  • 不仅适用于合作交互并且适用于涉及到物理层面和信息层面的竞争或混合（合作+竞争）环境

本文使用了集中训练，分散执行的框架，使用额外的全局信息来缓解训练的压力，但是在测试时不使用这些信息。因此，本文提出了一种actor-critic policy gradient方法的简单扩展，critic增加了其他智能体的额外信息，而actor只接触自身信息。在执行阶段，只使用local actor，以分散的方式执行。

MADDPG

Multi-Agent Actor Critic多智能体设计

MADDPG使用集中学习，分布式执行的策略，在学习中我们使用一些全局信息，而在应用的时候使用局部信息进行决策，Q-learning无法实现这一点，因为Q-learning在学习和应用时必须采用相同的信息。因此这里MADDPG对于传统的AC算法进行了改进，将Critic扩展为可利用其他智能体策略进行学习，即每个智能体对其他智能体的策略进行一个函数逼近。

我们使用θ=[θ1,...,θn]\theta = [\theta_1, ..., \theta_n]θ=[θ1​,...,θn​]表示nnn个智能体策略的参数，π=[π1,...,πn]\pi = [\pi_1, ..., \pi_n]π=[π1​,...,πn​]表示nnn个智能体的策略。之后我们可以将每个智能体iii的预期回报的梯度，J(θi)=E[Ri]J(\theta_i) = \mathbb{E}[R_i]J(θi​)=E[Ri​]为：

∇θiJ(θi)=Es∼pμ,ai∼πi[∇θilog⁡πi(ai∣oi)Qiπ(x,a1,…,aN)]\nabla_{\theta_{i}} J\left(\theta_{i}\right)=\mathbb{E}_{s \sim p^{\mu}, a_{i} \sim \pi_{i}}\left[\nabla_{\theta_{i}} \log \boldsymbol{\pi}_{i}\left(a_{i} \mid o_{i}\right) Q_{i}^{\pi}\left(\mathbf{x}, a_{1}, \ldots, a_{N}\right)\right] ∇θi​​J(θi​)=Es∼pμ,ai​∼πi​​[∇θi​​logπi​(ai​∣oi​)Qiπ​(x,a1​,…,aN​)]

在更新Loss的式子中，我们用到了其他智能体的策略，为了移除这部分的通信，每个智能体可以维持一个与智能体j的真实策略相关的近似函数μ^ij\hat{\mu}^j_iμ^​ij​。这个近似策略μ^ij\hat{\mu}^j_iμ^​ij​通过最大化智能体jjj的动作的对数概率加上策略熵学习（HHH为策略分布的熵）：

Agents with Policy Ensembles策略集合优化

多智能体强化学习中的一个反复的问题是由于智能体不断变化的策略而导致的环境非平稳性。竞争环境下尤其如此，因为智能体可以通过过度适应竞争对手的行为而获得很强的策略。
这样的策略是不可取的，因为它们很脆弱，当竞争对手改变策略时就可能会失败。

所以MADDPG提出了一种策略集合的思想，即第iii个智能体的策略μi\mu_iμi​由一个具有KKK个子策略的集合构成，在每一个episode中只随机使用其中一个子策略μi(k)\mu_i^{(k)}μi(k)​。对每一个智能体，我们最大化策略集合总体奖励目标：

Je(μi)=Ek∼unif⁡(1,K),s∼pμ,a∼μi(k)[Ri(s,a)]J_{e}\left(\boldsymbol{\mu}_{i}\right)=\mathbb{E}_{k \sim \operatorname{unif}(1, K), s \sim p^{\mu}, a \sim \mu_{i}^{(k)}}\left[R_{i}(s, a)\right] Je​(μi​)=Ek∼unif(1,K),s∼pμ,a∼μi(k)​​[Ri​(s,a)]

因为不同的子策略将在不同的博弈回合中执行，因此我们为智能体iii的每个子策略μi(k)\mu_i^{(k)}μi(k)​也维护一个replay buffer Di(k)D_i^{(k)}Di(k)​，更新方法为：

∇θi(k)Je(μi)=1KEx,a∼Di(k)[∇θi(k)μi(k)(ai∣oi)∇aiQμi(x,a1,…,aN)∣ai=μi(k)(oi)]\nabla_{\theta_{i}^{(k)}} J_{e}\left(\boldsymbol{\mu}_{i}\right)=\frac{1}{K} \mathbb{E}_{\mathbf{x}, a \sim \mathcal{D}_{i}^{(k)}}\left[\left.\nabla_{\theta_{i}^{(k)}} \boldsymbol{\mu}_{i}^{(k)}\left(a_{i} \mid o_{i}\right) \nabla_{a_{i}} Q^{\boldsymbol{\mu}_{i}}\left(\mathbf{x}, a_{1}, \ldots, a_{N}\right)\right|_{a_{i}=\boldsymbol{\mu}_{i}^{(k)}\left(o_{i}\right)}\right] ∇θi(k)​​Je​(μi​)=K1​Ex,a∼Di(k)​​[∇θi(k)​​μi(k)​(ai​∣oi​)∇ai​​Qμi​(x,a1​,…,aN​)∣∣∣​ai​=μi(k)​(oi​)​]

实验

本文实验环境为：grounded communication environment，由NNN个智能体和LLL个地标组成，存在一个具有连续空间和离散空间的二维世界。智能体在环境中执行物理动作并传递通信行为给其他智能体。一些环境能够完全显示通信而另一些环境只能执行物理动作。

实验环境源代码

Cooperative Communication 合作交流

由两个合作的智能体Speaker和Listener组成，Listener的任务是找到Sperker指定的地标。Listener可以观察地标的相对位置和颜色，但是不知道该去哪个地标，相反，Speaker的观察是正确地标颜色，以及由Listener观察的通信输出。

Predator-Prey 捕食者

N个较慢的合作智能体围绕一个随机生成的环境追赶更快的对手，并且有L个路障。合作智能体和对手发生冲突时，追捕者获得奖励，被捕者收到惩罚。

Cooperative Navigation 合作导航

智能体通过物理行动协作抵达L个地标。智能体观察其他智能体和地标的相对位置。接近地标获得奖励，相互碰撞获得惩罚。

Physical Deception 物理欺骗

N个智能体要合作到达N个目标地标，1个单个的对手也希望到达目标地标，任意一个智能体到达目标则获得奖励。
